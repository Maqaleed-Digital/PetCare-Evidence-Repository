# PETCARE DF21 — Promotion Approval Guardrails

## Purpose
Define hard boundaries for governed release promotion.

## Guardrails
- no promotion without DF20-approved policy context
- no promotion without explicit approval
- no promotion without ratification
- no promotion based on hidden logic
- no release activation without recorded baseline transition

## Evidence Basis
Release promotion must reference:
- DF20 policy_change_record
- DF20 version_transition
- DF19 feedback evidence when applicable
- active validated release candidate evidence

## Scope Guardrail
Promotion may be scoped to:
- global
- portfolio segment
- named unit

## Time Guardrail
Temporary promoted baseline must include expiry or re-ratification date.
