Runtime Control Gap Register Standard

Purpose

Define how runtime traceability and enforcement gaps are recorded, reviewed, and governed.

Gap Classes

RG-01
Missing implementation point

RG-02
Documentary-only control without runtime enforcement

RG-03
Partial implementation without validation

RG-04
Visibility evidence unavailable

RG-05
Rollback anchor undefined

Required Gap Fields

1. gap_id
2. control_id
3. gap_class
4. affected service
5. current status
6. owner
7. remediation path
8. target review date
9. closure evidence reference

Universal Rule

No gap may be suppressed or silently downgraded.
