# CONTROLLED_ACTIVATION_REPORTING_AND_ESCALATION

## Purpose
This document defines the minimum reporting and escalation requirements during controlled activation.

## Required reporting outputs
During activation, reporting must show:

1. gate decision status
2. activation sequence position
3. activation window assignment
4. first-day control status
5. incident count
6. override count
7. escalation count
8. hypercare entry status

## Escalation rule
Any material issue during controlled activation must be escalated immediately through the command model established in the readiness pack.

## Required outcomes
Each clinic under activation must be visible as one of:

- awaiting_gate_decision
- approved_for_activation
- activation_in_progress
- activation_issue_escalated
- hypercare_started
