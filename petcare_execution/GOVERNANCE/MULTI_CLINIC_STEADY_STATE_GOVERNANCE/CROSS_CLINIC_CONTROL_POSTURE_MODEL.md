# CROSS CLINIC CONTROL POSTURE MODEL

## Purpose
This document defines the minimum control posture that must remain active across all steady-state clinics.

## Required cross-clinic controls
The following controls are mandatory across the portfolio:

1. assistive AI boundary preserved
2. human approval enforcement preserved
3. override capture preserved
4. escalation routing preserved
5. KPI reporting preserved
6. partner dependency visibility preserved
7. portfolio comparison participation preserved
8. remediation re-entry path preserved

## Control rule
A clinic may be in steady-state only while all required controls remain governable and visible.

## Control posture states
Each clinic must be visible as one of:

- control_posture_compliant
- control_posture_watch
- control_posture_issue_open
