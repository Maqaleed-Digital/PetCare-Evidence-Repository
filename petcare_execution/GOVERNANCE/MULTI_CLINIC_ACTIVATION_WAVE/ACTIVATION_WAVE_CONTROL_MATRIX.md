# ACTIVATION WAVE CONTROL MATRIX

## Control domains
Each clinic wave is assessed across the following control domains:

1. Site Governance
2. Clinical Operations
3. Workforce Readiness
4. AI Safety and HITL Enforcement
5. Reporting and Evidence
6. Partner Connectivity
7. Escalation and Incident Routing
8. Hypercare Monitoring

## Wave progression rule
A clinic may only advance one stage at a time. Stage skipping is not allowed.

## Stages

### 1. Candidate Registration
Required:
- clinic code issued
- owner entity assigned
- activation wave assigned

### 2. Site Readiness Assessment
Required:
- readiness checklist completed
- staffing minimum confirmed
- local operating readiness confirmed

### 3. Pre-Activation Governance Review
Required:
- AI controls verified
- escalation path verified
- reporting path verified
- approval record created

### 4. Controlled Activation
Required:
- activation window approved
- clinic admitted under heightened observation
- first-day evidence capture enabled

### 5. Hypercare Monitoring
Required:
- override events reviewed
- exception events reviewed
- KPI capture reviewed
- human boundary compliance reviewed

### 6. Steady-State Admission
Required:
- hypercare exit approved
- baseline KPI variance acceptable
- portfolio reporting active

## Hard stop controls
Any of the following prevents activation:

- missing human approval enforcement
- missing override capture
- missing escalation routing
- missing evidence path
- missing KPI visibility
- unresolved material clinical safety issue
