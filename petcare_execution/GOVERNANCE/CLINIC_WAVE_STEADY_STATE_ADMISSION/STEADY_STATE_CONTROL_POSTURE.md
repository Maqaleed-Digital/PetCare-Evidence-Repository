# STEADY_STATE_CONTROL_POSTURE

## Purpose
This document defines the minimum control posture required after a clinic is admitted to steady-state.

## Required steady-state controls
The following controls remain mandatory after admission:

1. assistive AI boundary preserved
2. human approval enforcement preserved
3. override capture preserved
4. escalation routing preserved
5. KPI reporting preserved
6. partner dependency visibility preserved
7. portfolio comparison participation preserved
8. remediation re-entry path preserved

## Rule
Steady-state does not mean unguided operation. Governance remains active after admission.

## Drift rule
If a clinic materially drifts from required controls after admission, it may be reclassified for remediation review.
