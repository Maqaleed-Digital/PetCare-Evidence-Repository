# POST_HYPERCARE_REPORTING_AND_COMPARISON

## Purpose
This document defines the reporting and comparison outputs required after hypercare as clinics move toward or into steady-state.

## Required reporting outputs
Post-hypercare reporting must show:

1. steady-state admission status
2. last hypercare decision outcome
3. unresolved deviation count
4. override posture status
5. escalation posture status
6. KPI completeness status
7. partner fallback posture status
8. portfolio comparison status

## Required clinic states
Each clinic in this phase must be visible as one of:

- ready_for_admission_review
- admitted_to_steady_state
- remain_in_hypercare
- remediation_required
- post_admission_monitoring

## Portfolio comparison use
The portfolio view must support comparison between:
- baseline live clinic
- newly admitted steady-state clinics
- clinics remaining in hypercare
- clinics in remediation
