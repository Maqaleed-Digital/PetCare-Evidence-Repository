# HYPERCARE_EXIT_DECISION_FRAMEWORK

## Purpose
This document defines the decision framework for exiting hypercare.

## Exit inputs
Hypercare exit review must consider:

1. observation stability
2. unresolved deviation count
3. override posture
4. escalation posture
5. KPI completeness posture
6. partner fallback posture
7. portfolio command approval
8. steady-state readiness confirmation

## Exit outcomes
The only allowed outcomes are:

- approved_for_steady_state
- remain_in_hypercare
- return_to_remediation

## Decision rule
A clinic may exit hypercare only through explicit portfolio-level approval.

## Prohibition
No clinic may self-declare steady-state readiness.
