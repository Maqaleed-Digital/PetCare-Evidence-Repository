# HYPERCARE_REPORTING_AND_PORTFOLIO_REVIEW

## Purpose
This document defines the minimum reporting and portfolio review outputs required during hypercare.

## Required reporting outputs
During hypercare, reporting must show:

1. hypercare status
2. observation stability status
3. deviation count
4. override count
5. escalation count
6. KPI completeness status
7. partner fallback count
8. exit decision status

## Portfolio review use
The portfolio review view must support:

- comparison across hypercare clinics
- identification of unstable clinics
- decision support for steady-state admission
- prioritization of remediation review

## Required clinic states
Each clinic under hypercare must be visible as one of:

- in_hypercare
- hypercare_stable
- hypercare_issue_open
- ready_for_exit_review
- remediation_required
