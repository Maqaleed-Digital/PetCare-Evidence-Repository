# HYPERCARE ENTRY AND EXIT CRITERIA

## Purpose
This document defines the mandatory entry and exit conditions for newly activated clinics during the early supervised operating period.

## Hypercare entry criteria
A clinic enters hypercare only after:

- admission checklist passes
- activation approval is issued
- local command ownership is assigned
- first-day evidence capture is ready

## Hypercare monitoring focus
During hypercare, monitoring must explicitly cover:

1. override events
2. escalation events
3. KPI completeness
4. staffing exceptions
5. partner fallback usage
6. AI safety deviations

## Hypercare exit criteria
A clinic exits hypercare only when:

- no material unresolved safety issue exists
- override posture is within controlled tolerance
- escalation path is functioning
- reporting is complete
- portfolio comparison view is active
- command review signs off

## Hypercare decision outcomes
A clinic may end hypercare as:

- admitted_to_steady_state
- remain_in_hypercare
- return_to_remediation
