Scale Expansion Scope Model

Purpose

Define the governed scope patterns allowed under controlled scale expansion.

Scope Classes

SSM-01
Name
clinic_expansion

Description
Add a bounded number of clinics or operating units under existing governance, runtime, and ownership controls.

Required Controls
site scope definition
owner assignment
hypercare coverage
rollback path

SSM-02
Name
partner_expansion

Description
Add approved partner cohort under fairness, onboarding, and dispute governance.

Required Controls
partner cohort definition
fairness review
onboarding readiness
appeal and dispute readiness

SSM-03
Name
region_expansion

Description
Enter a bounded new operating region only under explicit governance and operational readiness review.

Required Controls
region boundary
operational capacity
runtime control mapping
rollback trigger rules

SSM-04
Name
service_expansion

Description
Activate a new bounded service class only when runtime enforcement and audit visibility are ready.

Required Controls
service class definition
readiness review
runtime mapping
restoration rule

SSM-05
Name
limited_scale_pilot

Description
Run a bounded pilot expansion with explicit expiry, review window, and restoration or progression path.

Required Controls
pilot scope
expiry
success criteria
restoration or progression rule

Universal Rule

No scope class may silently expand beyond its approved boundary.
