DF42 Approved Integrity Control Catalog

AIC-01
Name
Cross-layer conflict detection control

Allowed When
Used to identify conflicting rules, incompatible outcomes, or ambiguous precedence across governance layers

Required Controls
precedence ruleset
review owner
evidence trace
human-reviewed outcome

AIC-02
Name
Loophole chain detection control

Allowed When
Used to detect combined exception paths that could bypass stronger controls through multi-layer sequencing

Required Controls
conflict standard
layer chain evidence
approval reference
rollback posture

AIC-03
Name
Precedence lock control

Allowed When
Used to formally apply approved control precedence for a specific combined governance scenario

Required Controls
precedence decision
owner
approval reference
time-bounded scope

AIC-04
Name
Exception containment control

Allowed When
Used to keep approved exceptions narrow, reviewable, and non-propagating across unrelated governance domains

Required Controls
exception scope
expiry
review mode
restoration posture

Universal Rule

No integrity control may silently change precedence or activate unresolved conflicts without human-reviewed evidence.
