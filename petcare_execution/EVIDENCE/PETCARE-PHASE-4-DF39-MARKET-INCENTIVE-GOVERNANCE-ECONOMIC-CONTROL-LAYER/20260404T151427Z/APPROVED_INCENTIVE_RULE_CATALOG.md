DF39 Approved Incentive Rule Catalog

AIR-01
Name
Time-boxed partner onboarding support

Allowed When
Used to accelerate initial verified partner activation within a defined time window

Required Controls
approval reference
cohort definition
fixed cap
end date
fairness review

AIR-02
Name
Service quality performance bonus

Allowed When
Used to reward approved metrics such as SLA adherence, fulfillment accuracy, or documented quality outcomes

Required Controls
metric definition
measurement window
no hidden ranking boost
approval reference
rollback posture

AIR-03
Name
Emergency coverage support credit

Allowed When
Used to sustain approved emergency participation where coverage continuity is a declared platform objective

Required Controls
coverage rationale
bounded credit cap
time scope
fairness review across eligible network participants
approval reference

AIR-04
Name
Inventory stabilization support

Allowed When
Used to address approved supply imbalance, cold-chain readiness, or limited service continuity risks

Required Controls
documented operational rationale
bounded scope
review date
approval reference
no non-transparent partner favoritism

AIR-05
Name
Contracted revenue-share adjustment

Allowed When
A specific contract or approved commercial framework permits time-scoped economic variation

Required Controls
contract reference
share cap and floor
validity window
dispute path
approval reference

AIR-06
Name
Targeted pilot incentive

Allowed When
A pilot cohort is formally approved to test a new commercial structure without broader market distortion

Required Controls
pilot hypothesis
participant list
time-box
success criteria
rollback rule
approval reference

Universal Rule

No approved incentive may become permanent by default. Continued operation requires renewed review, fresh evidence, and explicit re-approval.
