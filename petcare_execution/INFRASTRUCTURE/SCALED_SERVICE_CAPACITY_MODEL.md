Purpose
Define the capacity model for supporting multiple clinics under live operations.

Capacity Dimensions
- application runtime capacity
- database capacity
- storage capacity
- support coverage capacity
- partner and integration capacity
- AI runtime usage capacity
- observability platform capacity

Required Outputs
- current baseline
- forecast by wave
- threshold risks
- scaling actions
- owner
- review date

Rules
- each expansion wave must reference the current capacity model
- threshold risks require actions before launch
- capacity assumptions must be reviewed before approval
