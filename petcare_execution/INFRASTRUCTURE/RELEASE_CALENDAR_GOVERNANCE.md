PETCARE-STEADY-STATE-OPERATIONS-AND-CONTINUOUS-IMPROVEMENT-GOVERNANCE
RELEASE_CALENDAR_GOVERNANCE

Purpose
Define the governed release calendar for steady-state operations.

Release Calendar Fields
- release_window
- release_type
- source commit hash
- approver
- deploy operator
- evidence path
- rollback readiness
- post-release verification status

Rules
- releases must follow named windows unless emergency change rules apply
- no anonymous approval
- no release without evidence linkage
- post-release verification status must be updated
- freeze windows must be respected
