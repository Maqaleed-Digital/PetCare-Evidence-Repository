CHANGE_FREEZE_AND_EMERGENCY_CHANGE_RULES

Purpose
Define the change freeze and emergency change policy during hypercare.

Change Freeze Rules
- non-essential production changes are restricted during active hypercare
- only approved fixes, safety changes, or blocking operational corrections are allowed
- every production change requires reason, owner, and evidence linkage

Emergency Change Rules
- emergency change requires named approver
- emergency change requires rollback readiness confirmation
- emergency change must be logged in daily review
- emergency change must not bypass audit evidence

Prohibited
- undocumented production changes
- anonymous approval
- unrelated feature work during active hypercare
- breaking AI assistive-only boundaries

Exit Rule
Freeze relaxes only after hypercare closeout or explicit governance approval.
