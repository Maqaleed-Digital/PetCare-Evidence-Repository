CAPACITY_AND_AVAILABILITY_PLANNING

Purpose
Define the steady-state capacity and availability planning baseline.

Planning Scope
- application runtime capacity
- database capacity
- storage growth
- observability platform capacity
- partner and clinic growth impact
- AI runtime usage growth

Minimum Planning Outputs
- current utilization summary
- forecast horizon
- threshold risks
- scaling actions
- owner
- review date

Rules
- capacity planning must be reviewed on a named cadence
- threshold risks require named actions
- availability risks must be highlighted before threshold breach
