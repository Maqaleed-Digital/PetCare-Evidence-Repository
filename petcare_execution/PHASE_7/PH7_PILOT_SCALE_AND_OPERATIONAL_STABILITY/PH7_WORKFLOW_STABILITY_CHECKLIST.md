PH7 WORKFLOW STABILITY CHECKLIST

S-01 PUBLIC AVAILABILITY
- home reachable
- signin reachable
- onboarding reachable
- unauthorized reachable

S-02 PROTECTED ACCESS INTEGRITY
- protected routes remain protected
- no raw JSON 403 in standard browser journey
- no route leakage

S-03 PILOT WORKFLOW REPETITION
- multiple pilot workflows completed
- owner role-safe access preserved
- vet role-safe access preserved
- admin oversight available if needed

S-04 CLINICAL CONTROL
- human sign-off preserved
- no sign-off bypass
- no wrong-role clinical action

S-05 INCIDENT VISIBILITY
- all observed incidents logged
- severity assigned
- owner assigned
- mitigation recorded if applicable

S-06 OPERATIONAL STABILITY
- repeated workflow evidence exists
- no uncontrolled scale
- notes captured for friction, delays, or instability

STOP CONDITIONS
- uncontrolled expansion beyond pilot cap
- use of demo or fake accounts
- hidden incidents
- sign-off bypass
- wrong-role access
- missing evidence for repeated workflow activity
